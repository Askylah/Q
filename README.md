## 🏛️ The Three Pillars

### 1. Immersive Roleplay & LARP
Run complex, multi-persona group chats with distinct namespaces. Whether it's a DnD campaign, a red-teaming scenario, or a custom fictional world, Q preserves state across every interaction.
*   **Persona Orchestration:** Mix and match models (Claude, Gemini, GPT) in a single room.
*   **Observer Mode:** Set personas to silently reflect on the conversation before contributing.
*   **Universal Identity:** Maintain a consistent avatar and name across all sessions.

### 2. Auto-Zettel Knowledge Graph
The sandbox doesn't just "chat"—it *remembers*. 
*   **Dynamic Lorebooks:** Automate world-building with a hybrid Vector + FTS5 memory system.
*   **Temporal Decay:** Observations and memories evolve or fade, mimicking organic knowledge.
*   **Relational Logic:** Link entities and events across a 1-hop knowledge graph.

### 3. Strict Physics (Security)
A deterministic execution engine built on Docker. Run untrusted code generated by agents in a root-jailed, network-isolated "Padded Room."
*   **Resource Clamping:** Hard limits on CPU, RAM, and PIDs.
*   **Temporal Guillotine:** Automatic SIGKILL for runaway processes.
*   **Governance Gates:** Human-in-the-loop approval for all file-system operations.

## 🚀 Quick Start

### Prerequisites
*   **Python 3.14+**
*   **Node.js & npm**
*   **Docker Desktop** (Required for the Workspace Engine)

### Setup
1. **Clone the repository.**
2. **Install Backend Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```
3. **Install Frontend Dependencies:**
   ```bash
   cd vite-project
   npm install
   ```

### Running the App
1. **Launch the Backend:**
   ```bash
   python main.py
   ```
2. **Launch the Frontend:**
   ```bash
   cd vite-project
   npm run dev
   ```
3. **Configure:** Access the **Settings** tab in the UI to set your API keys and select your models.
